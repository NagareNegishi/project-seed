# Build log

One entry per build session: `<yyyy-mm-dd>-<slug>.md`, committed with that
session's work. Written by the manager during a `build-orchestration` session.

Keep only what a later session needs:

- which option was chosen and why,
- decisions with their reasoning,
- how the built pieces connect to each other and to the plan,
- any finding accepted as risk.

Cut everything else — agent transcripts, play-by-play, restated plan content,
per-agent credit. If a line does not change what a future reader does, it is
noise. Entries are written through the `human-writing` skill.
